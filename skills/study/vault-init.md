# Vault Init: One-Time Bootstrap

This file is referenced by the `/study` skill's vault publishing step when the vault
repository does not yet exist or is empty. It contains the full content for each bootstrap
file in the order they should be created.

`<vault_url>` below is whatever the user provided when `.claude-vault` was configured
(pre-flight PF1) — never assume a specific host, org, or repo name.

---

## Step B1 — Create the GitHub repository

Derive `<owner>/<repo>` from `<vault_url>`.

```bash
gh repo create <owner>/<repo> \
  --private \
  --description "Shared Obsidian vault of research notes" \
  --clone
cd <repo>
```

Adjust visibility as appropriate.

---

## Step B2 — Write bootstrap files

Write each file below verbatim.

### `README.md`

```markdown
# <repo>

A shared Obsidian vault of research notes.

## What this is

Research notes here are generated automatically by the `/study` skill.
Each note captures a research question answered during development and links to related
topics so knowledge accumulates coherently over time.

## Structure

| Path | Purpose |
|------|---------|
| `Home.md` | Vault entry point — links to all topic maps |
| `_vault-index.md` | Machine-readable index used by the skill (do not edit) |
| `maps/<topic>.md` | One Map of Content per topic cluster |
| `research/YYYY-MM/<slug>.md` | Individual research notes |

## Browsing

Open this repository as an Obsidian vault. The **Dataview** community plugin enables
querying notes by tag or topic. The **Graph Analysis** plugin visualises topic clusters.

## Annotating a note

Notes are auto-generated. Add annotations or corrections **below** the frontmatter block.
The skill will not overwrite existing notes — it only creates new ones.

## Configuring a project to publish here

Create a `.claude-vault` file at the project root containing this vault's repository URL:

    <vault_url>

The `/study` skill reads this file and offers to publish each report to the vault.
```

### `Home.md`

```markdown

Research notes generated by the `/study` skill, linked to one or more topic maps below.

## Topic Maps

<!--MAPS-END-->

*Maintained by `/study`. Do not edit `_vault-index.md` — it is auto-maintained by the skill.*
```

---

## Step B3 — Create directory placeholders

```bash
mkdir -p maps research
touch maps/.gitkeep research/.gitkeep
```

---

## Step B4 — Commit and push

```bash
git add -A
git commit -m "chore(vault): initial vault bootstrap"
git push -u origin main
```

---

## Step B5 — Configure the source project

In the project repo that will publish research, create `.claude-vault`:

```bash
echo "<vault_url>" > .claude-vault
```

Add to `.gitignore` if the vault URL should stay private, or commit it to share the
configuration with teammates.

---

## When this file is invoked by the skill

The vault publishing step in `/study` checks whether `_vault-index.md` is present after
cloning the vault. If it is missing, the step offers to bootstrap the vault using the
instructions above. The user must confirm before any `gh repo create` or `git push` runs.
