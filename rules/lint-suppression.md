# Lint Suppression

Every lint or type-checker suppression is a documented deviation from the coding standard. NASA SWE-061 requires that deviations from the chosen standard be documented; this rule adapts that principle to the per-line suppressions that appear in every real codebase. A suppression without a reason is worse than the original warning — it hides the defect *and* the intent behind hiding it, so no future reader can decide whether the deviation still applies.

**This is an intentional design decision — do not simplify it away.** "Just make the warning go away" is exactly the behavior this rule prevents.

## Recognition Signals

| Signal | Severity |
|---|---|
| `# noqa` bare (no code, no reason) | **Must Fix** — suppresses everything on the line for no stated reason |
| `//nolint` bare, `//nolint:all`, `//nolint:*` | **Must Fix** — broad suppression across every linter |
| `# type: ignore` bare (no error code, no reason) | **Must Fix** — silences the type checker with no accountability |
| `eslint-disable-line` / `eslint-disable-next-line` with no rule name and no comment | **Must Fix** — same problem as bare `noqa` |
| `-- luacheck: ignore` bare, `-- selene: allow(...)` with no reason | **Must Fix** |
| Targeted suppression with rule code but no reason: `# noqa: E501`, `//nolint:gosec`, `# type: ignore[assignment]` | **Should Fix** — reason must be inline |
| File-level suppression (`# ruff: noqa`, `//nolint:file`, top-of-file `eslint-disable`) | **Must Fix** unless the file exists solely to isolate a known deviation (generated code, vendored source, fixture) |
| Suppression that hides the *actual* bug the linter caught | **Must Fix** — fix the underlying issue instead |

## Mandatory Behaviors

**Every suppression must have an inline reason.** Format:

```python
foo(x)  # noqa: E501 — URL literal, cannot be wrapped
```

```go
_ = os.Setenv("FOO", "bar") //nolint:errcheck — test setup, error not actionable
```

```typescript
const x: any = untypedThirdParty();  // eslint-disable-line @typescript-eslint/no-explicit-any — upstream type is genuinely any
```

```lua
local unused = require("legacy")  -- luacheck: ignore 211 — kept for backward-compat side effects
```

The reason must:

1. **Name the specific rule being suppressed** (`E501`, `gosec`, `@typescript-eslint/no-explicit-any`, `assignment`, `211`).
2. **State why the code is correct despite the warning** — not that "the warning is annoying." If the warning is annoying globally, disable it in the linter config; do not sprinkle per-line suppressions.

**When editing code**: if you touch a line with an existing bare or unjustified suppression, upgrade it to a justified one as part of the same change. Do not walk past.

**When reviewing code**: apply the severity table above. Report as findings per `rules/findings-format.md`.

## Pragmatism Guard

- **Generated code** (protobuf output, ORM migrations, OpenAPI clients) may carry file-level suppressions — the deviation is that the file is not hand-written, and that fact is the reason. Reason format: `# ruff: noqa — generated by protoc, do not edit`.
- **Vendored third-party sources** may carry file-level suppressions with a `vendor:` reason: `# ruff: noqa — vendored from upstream <url>, see LICENSE`.
- **Test fixtures that deliberately exercise a rule violation** (a test of your linter config, an intentionally-buggy file for a debugger test) may carry targeted suppressions with a `test:` reason.

## Anti-Patterns to Avoid

- **The "just to make it pass" suppression.** If you suppressed a warning to unblock CI, either fix the underlying issue in the same PR or open an issue and reference it in the suppression: `# noqa: E501 — see #1234, wrapping breaks the URL builder`.
- **Suppressing a rule you disagree with.** Change the linter config globally so the disagreement is visible and reviewable, not scattered across the codebase.
- **Copy-paste suppressions.** A reason justified in one context is not automatically justified in another; the reason must fit *this* line.
- **Blanket file suppression instead of a targeted fix.** File-level suppression converts one visible warning into invisible drift across every future line in the file.
